# Example: 工作中做了很多但力量没有叠加

## 输入

```text
/yis-loop

我有个同事每天都很忙，开会、回消息、改文档、接新需求。
但我感觉他没有形成真正的成果。
他也常说自己很累，但看不到积累。
我想帮他找到重复力量的点。
```

## 可能输出

```text
# YIS Loop Diagnosis

## Your Current Scattered State
这个同事的问题不是不努力，而是动作大多停留在响应层。
他每天都在处理输入，但没有固定输出被持续叠加。

## Dispersion Pattern
- Pattern: result panic + identity switching
- Evidence: 他被会议、消息、临时需求牵引，缺少一个每天都能留下成果的核心动作。

## Your Core For The Next 7 Days
For the next 7 days, repeat "每天产出一个可复用的工作成果块" to strengthen "从被动响应到主动积累的能力", using "这个成果明天能否复用" after each loop.

## The 7-Day Loop
1. Repeat: 每天完成一个可复用成果块，如模板、清单、决策记录、流程说明。
2. Record: 记录这个成果服务了哪个重复问题。
3. Adjust: 第二天优化同一类成果，而不是换一个新方向。

## Do-Not-Do List
- 不把回消息当成成果。
- 不把开会当成进展。
- 不同时优化 5 个工作方向。
- 不用“今天很忙”证明自己有效。

## One-Line Prescription
忙不是力量，能留下并复用的东西才是力量。
```
